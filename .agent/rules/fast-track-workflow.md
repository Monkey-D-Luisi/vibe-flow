# Fast Track Workflow

## When to Use
Meta-work, documentation, urgent fixes, process improvements.

## Naming
`ft-NNNN-<short-slug>.md`

## Execution Contract

Present a brief plan and get user approval before implementing. For any doubt, use the questionnaire tool (always include a free-text field).

## Workflow
1. Create `docs/tasks/ft-NNNN-<slug>.md`
2. Create `docs/walkthroughs/ft-NNNN-<slug>.md`
3. **Plan & Confirm**: Outline the approach (files to change, key decisions) and wait for user approval
4. Implement following quality standards
5. Commit: `docs(scope): description (#ft-NNNN)`
6. Update walkthrough
