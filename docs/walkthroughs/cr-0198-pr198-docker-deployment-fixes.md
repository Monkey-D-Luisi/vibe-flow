# CR-0198 — PR #198 Docker Deployment Review Fixes — Walkthrough

## What was done

Processed 13 inline review comments from Gemini Code Assist and GitHub Copilot on
PR #198 (feat: docker deployment configuration for EP08 autonomous product team).

## MUST_FIX resolutions

### M1: Pinned pnpm version in Dockerfile
`pnpm@latest` is non-reproducible — the installed version changes over time and can
break `--frozen-lockfile` installs when the lockfile was generated by a different
major. Pinned to `pnpm@10.18.1` (the current local version, matching lockfileVersion 9.0).

### M2: Fixed ENTRYPOINT
`npx openclaw` would download an arbitrary version from npm registry at runtime.
Changed to `["pnpm", "exec", "openclaw", "gateway", "start"]` so the workspace-installed
binary is used deterministically.

### M3: Removed non-existent plugin load paths
`telegram-notifier`, `stitch-bridge`, `model-router`, and `team-ui` directories do not
exist in this PR (they are tracked as separate tasks 0037-0041). Leaving their paths in
`plugins.load.paths` would cause the gateway to fail at startup. Removed those 4 paths;
kept their `plugins.entries` configs since they are harmless when the plugin is absent.

### M4: Disabled CI feedback webhook
`ciFeedback.enabled: true` with `webhookSecret: "${GITHUB_WEBHOOK_SECRET:-}"` (empty
default) causes the product-team plugin to throw on startup when the secret is blank.
Disabled for now (`enabled: false`). CI feedback will be wired up in Task 0037.

### M5: Added missing env vars to .env.docker.example
`openclaw.docker.json` references `GITHUB_OWNER`, `GITHUB_REPO`, and expects
`GITHUB_WEBHOOK_SECRET` to be defined. Added all three to the example template with
clear comments.

## SHOULD_FIX resolutions

### S1: Port binding restricted to localhost
Changed `"28789:28789"` → `"127.0.0.1:28789:28789"` in `docker-compose.yml`. With
`auth.mode=none` in the gateway config, binding to `0.0.0.0` would make the
unauthenticated control UI reachable from any host on the network. Localhost-only
binding is safe for local development.

### S2: EP08 backlog status aligned to IN_PROGRESS
`docs/backlog/EP08-autonomous-product-team.md` had `Status: PENDING` while
`docs/roadmap.md` correctly reflected `IN_PROGRESS`. Aligned to `IN_PROGRESS`.

### S3: Task spec D2 example updated
The D2 compose snippet in `docs/tasks/0035-docker-deployment-config.md` did not match
the implemented `docker-compose.yml` (had `./secrets:/app/secrets:ro` volume and
`OPENCLAW_GATEWAY_PORT`/`OPENCLAW_GATEWAY_BIND` env vars). Updated to reflect the actual
implementation using `env_file` + `127.0.0.1` port bind.
