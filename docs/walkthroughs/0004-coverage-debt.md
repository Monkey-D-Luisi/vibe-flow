- Epic: Pre-EP04 prerequisite

- PR: TBD

## Summary

Raised `extensions/product-team` file-level coverage above the task threshold by
adding focused tests for uncovered branches in plugin registration, task update
no-op behavior, step runner edge paths, and transition guard fallbacks.

Also fixed the coverage tooling precondition by adding `@vitest/coverage-v8` to
`@openclaw/plugin-product-team` dev dependencies so `pnpm test:coverage` can run.

---

## Context

- Baseline test suite was green (`147` tests) but file-level coverage had gaps
  below `80%` in key files.
- Initial `pnpm test:coverage` failed because `@vitest/coverage-v8` was missing.
- Task constraints required test-focused changes (no production behavior changes).

---

## Decisions

| Decision | Rationale |
|----------|-----------|
| Add `@vitest/coverage-v8` as a dev dependency | Coverage command was failing before any measurement could happen |
| Target only low-coverage files with narrow tests | Fastest path to deterministic coverage recovery without refactors |
| Add a direct `runWorkflowSteps` unit suite | Existing tool-level tests did not hit several internal branch paths |

---

## Implementation Notes

### Approach

1. Unblocked coverage runner (`@vitest/coverage-v8`).
2. Measured baseline and identified files under threshold.
3. Added targeted tests per uncovered branch.
4. Re-ran coverage and quality gates at package and workspace levels.

### Key Changes

- `test/index.test.ts`
  - Added coverage for:
    - non-string `dbPath` fallback (`:memory:`)
    - workspace path-escape rejection
    - shutdown handler registration/invocation
    - executing a registered `task.create` tool
- `test/tools/task-update.test.ts`
  - Added no-op branch tests when no mutable fields are passed:
    - returns current task
    - throws not-found for unknown task
- `test/orchestrator/step-runner.test.ts` (new)
  - Added direct unit tests for:
    - missing task error path
    - script step branch with non-object output normalization
    - non-`Error` validation failure wrapping
- `test/orchestrator/transition-guards.test.ts`
  - Added missing/invalid metadata guard cases for `design`, `in_progress`,
    `in_review`, and `qa` transitions
  - Added config fallback tests for malformed guard configuration
