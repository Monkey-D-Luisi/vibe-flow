A human posts an idea in Telegram. The PM creates a roadmap item and breaks it into epics. The PO refines each epic into user stories with acceptance criteria. The Tech Lead decomposes stories into technical tasks and assigns them. The Designer creates mockups via Google Stitch. Backend and Frontend Devs implement in parallel with TDD. QA validates with automated test suites. The Tech Lead reviews the code. DevOps ships the PR and manages CI. All status updates broadcast to the Telegram group in real time.

Modular Extension Architecture

5 extensions compose the system. Each is a self-contained OpenClaw plugin with its own tools and responsibilities.

product-team

34+ tools

The core engine. Task lifecycle, workflow execution, quality gates, VCS automation, team messaging, decision engine, and pipeline orchestrator.

quality-gate

CLI

Standalone quality enforcement. Tests, coverage, lint, complexity -- all without task-engine state. CLI-first for local and CI pipelines.

model-router

Hook

Per-agent model routing. Each agent gets the right LLM: GPT-5.3 for strategy, Opus 4.6 for architecture, Sonnet 4.6 for code.

telegram-notifier

Bridge

Human oversight via Telegram. Real-time updates on task transitions, PR links, quality gates, and decision escalations. Observe or intervene.

stitch-bridge

MCP

Google Stitch MCP integration. The designer agent creates visual mockups before coding begins. Design-driven development pipeline.

Built for Production

Not a prototype. Battle-tested through 9 epics, 6 audits, and 119+ autonomous tasks.

Hexagonal Architecture

5-layer domain model with ports and adapters. Clean boundaries between tools, domain, persistence, and orchestration.

State Machine with Guards

Transition guards enforce evidence: architecture plan, dev metrics, review result, QA report. No shortcuts.

SQLite Persistence

Event-sourced task records with append-only log, lease-based ownership, and WAL mode for concurrency.

Quality Gates

Tests, coverage (≥80%), lint (zero errors), complexity (≤5.0), vulnerability scanning. Enforced at every transition.

Decision Engine

6 decision categories with configurable policies: auto-resolve, escalate to tech lead, pause for human, retry with backoff.

Inter-Agent Messaging

Direct agent-to-agent communication with auto-spawn, event logging, and Telegram forwarding for urgent messages.

GitHub Automation

Idempotent branch creation, PR management, label sync, CI webhook feedback. All via gh CLI wrapper.

Docker Deployment

Single-container deployment with persistent volumes, health checks, and multi-project workspace support.

Multi-Model Routing

3 providers (OpenAI, Anthropic, Google), per-agent primary + fallback chains, provider health monitoring.

Pipeline Orchestration

10-stage pipeline with automatic advancement, stage timeouts, retry limits, and dead-letter alerting.

14 Role Skills

Markdown prompt libraries for every role: from requirements grooming to TDD implementation. Schema-validated outputs.

CI/CD Integrated

4 GitHub Actions workflows: CI, quality gate PR comments, npm publish with provenance, project board sync.
