An 8‑agent autonomous product team that ships through a 10‑stage pipeline.

vibe-flow is a monorepo of OpenClaw extensions, skills, and quality tooling. It runs a full product team inside an AI gateway — taking ideas from chat to merged PRs.

No frameworks. No build step. Mermaid diagrams. System fonts.

Why developers care
  • Role clarity: PM → PO → TL → Design → Dev → QA → Review → Shipping.
  • Guarded state machine: transitions require evidence, not vibes.
  • Auditability: decisions, events, and outcomes are logged.
  • Composable architecture: 5 extensions, clean responsibility split.

Meet the autonomous team (8 agents)

The 10‑stage pipeline

A predictable assembly line: each stage produces evidence that unlocks the next.

Stages

  1. 1 IDEA
  2. 2 ROADMAP
  3. 3 REFINEMENT
  4. 4 DECOMPOSITION
  5. 5 DESIGN
  6. 6 IMPLEMENTATION
  7. 7 QA
  8. 8 REVIEW
  9. 9 SHIPPING
  10. 10 DONE

Each stage posts updates back to the Telegram group — so humans can supervise without micromanaging.

End‑to‑end flow (from EP08)

flowchart TD A[Human posts idea in Telegram] --> B[PM receives idea → creates roadmap item] B --> C[PM breaks roadmap item into epics] C --> D[PO refines into user stories + acceptance criteria] D --> E[Tech Lead decomposes into technical tasks] E --> F[Designer creates Stitch designs] F --> G[Tech Lead assigns tasks to dev agents] G --> H[Devs implement (Backend & Frontend in parallel)] H --> I[QA runs automated test suites] I --> J[Tech Lead reviews code] J --> K[DevOps creates PR, manages CI] K --> L[All status updates → Telegram group]

5 extensions, one cohesive runtime

The landing page you’re reading is just static files — but the system behind it is modular and composable.

product-team

Task engine, workflow, quality, VCS automation, decision engine, pipeline orchestration.

quality-gate

Standalone quality engine + CLI for local and CI runs.

telegram-notifier

Lifecycle → Telegram notifications & routing.

model-router

Per-agent model routing via config + hooks.

stitch-bridge

Bridge to Google Stitch MCP for design tooling.

Tooling that makes autonomy practical

The product-team extension exposes a tool surface that agents can use safely: tasks, workflow steps, quality gates, VCS automation, team messaging, decisions, and pipeline orchestration.

Tool categories

  • Task lifecycle: task.create, task.transition, task.update, …
  • Workflow evidence: workflow.step.run, workflow.state.get, …
  • Quality: quality.tests, quality.lint, quality.gate, …
  • VCS: vcs.branch.create, vcs.pr.create, …
  • Team: team.message, team.assign, team.inbox, …
  • Decisions: decision.evaluate, decision.log, …
  • Pipeline: pipeline.start, pipeline.advance, pipeline.metrics

Source: docs/api-reference.md

Why it works

  • Guard rails: transitions enforce evidence requirements.
  • Audit trail: decisions and events are queryable.
  • Composable: extensions layer cleanly without monolith glue.

Quick start (Docker)

Run the autonomous team locally via the OpenClaw gateway + extensions bundle.

Start

Shell 
cp .env.docker.example .env.docker
# edit .env.docker
docker compose build
docker compose up -d

docker exec openclaw-product-team pnpm exec openclaw models list
docker exec openclaw-product-team pnpm exec openclaw doctor

Source: docs/docker-setup.md

What you get

  • Control UI: http://localhost:28789/
  • 8 agents + 5 plugins loaded
  • Telegram integration for human oversight
  • Quality gate CLI for local/CI validation

Tip: set LOG_LEVEL=debug when tuning cache hit/miss behavior.
